iniciando con git

GIT es un control de versiones que permite a equipos de desarrolladores trabajar de manera remota sobre un mismo proyecto. permite hacer contribiciones de codigo de manera local y luego replicarlos a un repositorio remoto.

principales comandos con GIT

  • git init: funciona para inicializar un proyecto en git
  • git config --global user.email "email": para identificar y configurar globalmente el email con el repositorio local
  • git config --global user.name "name" : funciona para identificar globalmente el nombre de usuaruio con el repositorio global
  • git config user.email: identifica el email agregado
  • git config user.name: identifica el nombre de usuario agregado
  • git add . : funciona para agregar todos los archivos modificados y nuevos en el repositorio local de git.
  • git status: permite verificar el estado del repositorio local.
  • git commit -m "comentario": permite guardar los cambios realizados en el repositorio local con un mensaje resumido de los cambios.
  • git log : muestra el historial de cambios en el repositorio local.
  • git branch: muesrtra la rama del repositorio local.
  • git checkout -b nombre_rama: para cambiar el nombre de la rama en un repositorio local.
  • git checkout -- . : deshacer los cambios de un archivo no confirmado.
  • git remote add origin url : el repositorio local se agrega a un repositorio remoto como GITHUB.
  • git push -u origin main: enviar los commits del repositorio local al repositorio remoto.hola
